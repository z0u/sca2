description: Markdown-specific formatting conventions for this project's docs and notebooks. Use alongside the writing skill, which owns voice, tone, and concision.
---

Voice, tone, hedging, and concision are the `writing` skill's job — load it for
any substantial prose. This skill adds only the Markdown mechanics:

Use sentence case for headings and descriptive lists.

```patch
- # Experiment Design
-
-   - **Foo Bar:** baz
+ # Experiment design
+
+   - Foo bar: baz
```

Prefer paragraphs for nuanced or complex explanations; use lists for
summarizing steps, or when clarity would genuinely benefit from structure —
and use them sparingly.
